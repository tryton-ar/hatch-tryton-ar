===============
hatch-tryton-ar
===============

This package provides a metadata hook plugin for Hatch_ that extends the 
functionality of hatch-tryton_ to automatically manage dynamic dependencies 
for the Argentine localization of Tryton.

It dynamically converts dependencies using the ``tryton-ar`` organization prefixes 
into direct, anonymous URLs pointing to their respective GitHub repositories, 
matching the branch name with the current module's major and minor version 
(``major.minor``).


Configuration
=============

The metadata hook plugin is registered under the name ``tryton-ar``.

To use it, ensure it is included in the build environment requirements (``build-system``) 
of your ``pyproject.toml`` along with the Hatch backend:

.. code-block:: toml

    [build-system]
    requires = [
        "hatchling >= 1",
        "hatch-tryton-ar @ git+git://github.com/tryton-ar/hatch-tryton-ar.git"
    ]
    build-backend = "hatchling.build"

Since ``hatch-tryton-ar`` internally depends on ``hatch-tryton``, the official plugin 
will be automatically installed and executed transparently in the build chain.


Configuration Options
---------------------

The configuration block is defined under the ``[tool.hatch.metadata.hooks.tryton-ar]`` section. 
Currently, the plugin does not require any additional parameters, as it directly intercepts 
the dependencies dictionary previously generated by the base plugin.

Full Workflow Example
---------------------

Below is a typical integration example for a module such as ``bank_ar``:

.. code-block:: toml

    [project]
    name = "trytonar_bank_ar"
    dynamic = ["version", "dependencies", "optional-dependencies", "authors", "readme"]

    [tool.hatch.metadata.hooks.tryton]
    config = "trytond/modules/bank_ar/tryton.cfg"
    readme = "README.rst"
    copyright = "COPYRIGHT"

    [tool.hatch.metadata.hooks.tryton.prefixes]
    party_ar = "trytonar"
    account_ar = "trytonar"

    # Tryton-AR plugin activation
    [tool.hatch.metadata.hooks.tryton-ar]


How it works
============

1. The official ``hatch-tryton`` plugin reads the ``tryton.cfg`` file and the ``prefixes`` section.
2. If it finds that the module depends on ``party_ar``, it generates a standard PyPI constraint:
   ``trytonar_party_ar >= 8.0, < 8.1``.
3. Immediately after, the ``tryton-ar`` hook intercepts this list, detects the ``trytonar_`` prefix, 
   extracts the current branch version (e.g., ``8.0``), and dynamically rewrites the entry to:
   ``trytonar_party_ar @ git+git://github.com/tryton-ar/party_ar.git@8.0``.

As a result, any installation command (such as ``pip install .``) will clone and install 
sibling modules from the organization anonymously, securely, and in a 100% automated manner.
